# Network2 — Phishing Detection & Training Pipeline

This repository contains a machine-learning pipeline for network/phishing detection built as a modular Python package (`networksecurity`). It provides components for data ingestion, validation, transformation, model training, and a FastAPI-based prediction endpoint.

**Key Features**
- End-to-end training pipeline runnable from `main.py`
- FastAPI service for training and prediction in `app.py`
- Artifacts and MLflow run logs under `Artifacts/` and `mlruns/`
- Prebuilt Dockerfile for containerized deployment

**Repository Structure (high level)**
- `main.py` — run the training pipeline (data ingestion → validation → transformation → model training)
- `app.py` — FastAPI app exposing `/train` and `/predict` endpoints
- `networksecurity/` — package with pipeline components, configs, and utilities
- `Artifacts/` — generated artifacts (ingested data, transformed arrays, trained models)
- `mlruns/` — MLflow experiment runs and metrics
- `final_model/` — place for packaged model and preprocessor
- `requirements.txt` — Python dependencies
- `Dockerfile` — container image specification
- `test_mongodb.py` — example/test for MongoDB connectivity

Getting started
1. Create a Python 3.8+ virtual environment and install dependencies:

```bash
python -m venv .venv
source .venv/Scripts/activate   # Windows: .venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

2. Create a `.env` file in the project root with at least the MongoDB connection key used by the app:

```
MONGODB_URL_KEY=<your_mongo_connection_string>
```

Running the training pipeline (local)

```bash
python main.py
```

This executes the pipeline defined in `main.py` which will create artifacts under `Artifacts/` and log runs in `mlruns/`.

Running the API server (prediction + trigger training)

```bash
python app.py
# or run via uvicorn: uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

Open the interactive docs at `http://localhost:8000/docs` to test the `/train` and `/predict` endpoints. The `/predict` endpoint accepts a CSV upload and returns an HTML table of predictions.

Data and artifacts
- Place raw dataset(s) under `Network_Data/` or use the ingestion component. Example dataset: `Network_Data/phisingData.csv`.
- Pipeline outputs (ingested CSVs, transformed numpy arrays, models) are stored in `Artifacts/<timestamp>/...`.

Limitations
- The prediction endpoint depends on `final_model/preprocessor.pkl` and `final_model/model.pkl`; these files are not automatically generated by the repo.
- There is no production-grade error handling, authentication, or request validation in the FastAPI app.
- The training pipeline is built for a specific phishing dataset schema and may require adaptation for new data sources.
- No automated unit or integration tests are included for pipeline components today.
- Docker support is optional and may require environment-specific changes before deployment.

Docker (optional)

Build and run the image (example):

```bash
docker build -t network2 .
docker run -p 8000:8000 --env-file .env network2
```

Testing
- Basic test/example: `python test_mongodb.py` (ensure `.env` is set with MongoDB URL)
